Order selection groups

The first thing you need to do to be ready to order stock is to define your order selection groups. Order selection groups allow you to create orders automatically because the groups eliminate the labor-intensive task of manually entering individual part numbers.

Add sample screens from the Resources folder; then add steps or topics on creating OSG for each order type -- one for Factory, NAPA, non-NAPA, and Special.

You can define as many selection groups as you require for each order type. The order types are:

  • Factory

  • NAPA

  • Non-NAPA

  • Special

For information about order types, see Order types.

After you select an order type, you define the parts to be ordered by specifying which suppliers, classification codes, product lines, and part numbers to include in the order. You also set the ordering logic that is used when the order is placed. (In RPM, order selection groups were known as order templates.)

Settings common to all order types

The Order selection group screen changes, depending on the order type that you are creating orders for. The following fields must be completed for all order types:

  • Include items flagged as hazardous Inventory that contains environmental hazards is flagged as hazardous. Some examples include NAPA batteries, oil, or antifreeze. Select this checkbox to include these items when this order selection group is used.

  • Include displayed merchandise Select this checkbox to include parts with a display quantity in the order.

  • Order based on

    This setting determines the logic used to calculate how many to order.

    Option

    Description

    Min/Max

    Keeps enough parts on the shelf to maintain optimal stock levels without overstocking. orders a part if the total quantity of on hand, on order, on back order, on factory order, minus work in progress is less than the minimum stocking level set for the part.

    When parts reach a specified minimum level, they are ordered up to the maximum level.

    For example, your Min is set to 5, your Max is set to 10, and you have 4 available after today's sales. When your order is calculated, 6 parts will be ordered to bring you up to the maximum of 10.

    Sales history

    Uses the months you specify to determine how much to order. This method is recommended for seasonal items, such as deicers in winter months or carwashing items in summer months.

    If you base your order on Sales history, click Select sales months and indicate which months to use to calculate the order amount.

    Days supply

    Calculates the quantity needed for a specific number of days. uses the average daily sales to calculate how many items you will need for the number of days that you specify.

    In addition, enter a value for Minimum demand. This number determines what minimum sales history (demand) a part must have before it is considered for ordering. For example, if you enter 5, a part has to have at least 5 sales in the time period designated before reviews it for potential ordering.

  • Automatic finalization Specifies an upper and lower dollar limit to determine when purchase orders are finalized automatically. Orders with totals within these limits are finalized automatically; orders that total less or more than these limits must be finalized manually. Enter a minimum and maximum order value to use automatic finalization. If you leave the default amounts of 0.00, all orders for this selection group are finalized automatically.

Maintain order selection groups

The Order Selection Group screen changes, depending on the order type that you select.

  1. From the menu, go to Inventory > Purchase orders > Maintain order selection group.

  2. Select an Order type, and then select New.

    See Order types to learn about the order types available.

  3. Enter an identifier in the Order selection group field, and then enter a name for the group in Alias.

  4. Complete the Settings common to all order types.
