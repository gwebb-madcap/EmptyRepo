Inventory detail and product line detail
Column What it shows

Line

Cash for the office.

Part

Part number. This column is blank under the Product Line Detail section.

Group code

Group code number. This will print in the Part column, under the Product Line Detail section.

CLS

Classification code. This column is blank under the Product Line Detail section.

Action

  

Emp

Employee's number who is associated with the record being printed.

Field affected

The field name that is affected by the action made to the record.

Before

The original value of the part, before any changes were done

After

The current value of the part, after changes were applied

On-hand quantity adjustment summary
Column What it shows

Line

Cash sales for the office.

Part

Part number.

Description

Name of part.

Cls

Classification code.

Emp

Employee's number who is associated with the record being printed.

Change Made By

The name of the TAMS II internal process where the adjustment was made. For example: FrmCLS_Finalize = Finalized classification.

On-Hand Before

The original on-hand value of the part, before any changes were applied.

On-Hand After

The current on-hand value of the part, after changes were applied.

On-Hand Adjusted

On-Hand Before minus On-Hand After values.

Cost

Inventory Cost + Inventory Core Cost. For returned cores on-hand, the printed cost is only core cost.

Amount

Sum of adjustment amount x cost.

Total for Line

Sum of all Amount fields for a product line.
On-Hand Quantity Adjustment Recap
Column What it shows  

Change Made By

The name of the TAMS II internal process where the adjustment was made. For example, FrmCLS_Finalize = Finalized classification.

  

Recap - Amount

Sum of the extended adjustment amount for each on-hand adjustment.

  

Recap - Total Adjusted

Sum of all Recap- Amount values, which equals the Total Inventory Adjustment value.

  

Landed Text

Sum for all cash payments received for all employees.
