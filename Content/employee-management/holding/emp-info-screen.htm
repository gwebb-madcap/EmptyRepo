Employee information screen

Use the Employee information screen to add new employees or maintain employee information.

Only employees with a manager-level code assigned in the Job code field have access to the Employee information screen.

Main tab

Use the Main tab to add a new employee or maintain employee information.

Field

Description

Employee

To edit information for an existing employee, enter the employee's number or select the employee from the list.

Employee number

Required. Must be between 0 and 9,999 and not in use.

Job code

Required. Indicates the employee's role. Only employees with manager-level job codes can access the Employee information screen.

Last name

Required. Allows up to 20 characters.

First name

Required. Allows up to 20 characters.

Middle initial

The first letter of the employee's middle name.

Birth date

Employee's date of birth in MMDDYYYY format.

Address 1,2

Up to two lines for the employee's home street address.

City

City in which the employee lives.

State

The abbreviation and name of the state in which the employee lives. This value determines what state income taxes are paid by the employee. Is this true even if TAMS is not used for payroll?

Zip

The employee's postal code.

Country

The country in which the employee lives (United States, Canada, or Mexico). This value determines U.S. federal or other national income taxes paid by the employee. Is this true even if TAMS is not used for payroll?

Phone number

The employee's primary telephone number, including area code. Hyphens are optional.

E-mail address

The employee's email address, up to 50 characters.

Pay type

Required. Indicates if the employee is hourly or salaried.

If time tracking is in use in , salaried employees are not required to clock in and out.

Profile number

Required if the Pay type is hourly. Indicates the pay period profile for the employee.

Language

The language (English, French, or Spanish) in which the employee views screens and messages. English is the default.

Invoicing allowed

Provides the employee access to the Point of Sale (Invoicing) screens when selected.

Display gross profit and cost

only; this field is not displayed for .

 

Select the checkbox to allow the employee to view gross profit and cost information in on Inventory Review screens and reports, Invoicing/Stock Check screens, and View Today's Sales screens. The default setting is not selected.

 

When the field is not selected, the following components are affected:

Catalog preference

Identifies the catalog that is displayed when an employee selects the Catalog button or presses F9 on the Invoice screen. This field overrides the Catalog preference field on the Invoicing controls screen.

recommends that you set the field to PartsPRO SE.

Store dept

The department in which the employee works, such as Counter or Manager.

A/R customer #

If each of your employees has an individual A/R customer number to use when purchasing parts, enter the employee's A/R customer number in this field.

 

This field affects the employee's ability to self-invoice.

Timeout minutes

The number of minutes of inactivity before automatically locks the screen. At that time, the employee must enter a password to continue.

 

The Employee access validation field on the System management > Store controls > General tab controls this field:

  • When Employee access validation is selected, Timeout minutes must be set from 1 to 10 for assistant manager, district officer, store manager, and hourly key holder employees. Timeout minutes can be set from 0 to 999 for other employees.

  • When Employee access validation is not selected, Timeout minutes can be set from 0 to 999 for all employees.

: Employee access validation is selected and cannot be changed.

: Employee access validation can be selected or cleared.

Employee password

Required. The employee's password used to log in, unlock the screen, or access any protected functions. When the field is set or changed on this screen, the employee is prompted to change the password upon logging in.

Expires in days

Number of days before the employee's password expires and must be changed. The employee receives a message to change the password 15 days before it expires.

The Employee access validation field on the System management > Store controls > General tab controls this field:

  • When Employee access validation is selected, Expires in days must be set from 1 to 90.

  • When Employee access validation is not selected, Expires in days can be set from 1 to 999 for all employees.

: Employee access validation is selected and cannot be changed.

: Employee access validation can be selected or cleared.

Employment date

Date the employee was hired (MMDDYYYY).

Service date

Date on which the employee becomes eligible for benefits (MMDDYYYY).

Termination date

Employee's last official work day (MMDDYYYY) for terminated employees.

 

When the Termination date is today or in the past, the employee's access to all functions is removed. The Invoicing allowed checkbox is cleared, and access to all functions is removed on the Security tab.

Pay rate

Employee's payroll rate, either a salary amount for a single pay period or the employee's regular hourly rate.

Employee ID

Specifies an ID used to track when the employee signs up a new NAPA Rewards member. The employee receives a spiff amount (varies per district) for each new member that signs up for NAPA Rewards.

This field is visible only when NAPA Rewards is turned on or the IBS portal is turned on in the store. The system validates if the ID is assigned to another employee and prompts you to enter a new ID if it is.
