From GWinko doc

  1. Impact of Posting PO’s without COM150

User will not be able to print OS&D report at time of posting of the PO or PO’s.

If COM150 is received after PO or PO’s are posted, COM150 will reflect “Above part does not affect stock quantity”- for any parts posted on the PO or PO’s. This is an indication to store on hand quantities will not be affected and a trigger to the user to print out the OS&D report.

  1. Scanning and get message that part is not in the session/PO

If a part is scanned while scanning in freight and a message appears stating that the part is not in the session / PO and asks if they want to add it to the session PO, that part should be set aside and dealt with separately.

The reason for this is that if that part is in a PO that contains other parts, if the person scanning in the freight tells the scanner to add the part to the session / PO, the entire PO containing the part that was scanned and prompted the message will be added to the session / PO being scanned in.

The business impact of this is that the remaining parts on that PO would have no received quantity and if the person editing and posting that Check-In Group or PO doesn’t notice this, they would prompt the system to post those parts as none received, which would clear on order information and potentially make those parts eligible for ordering again. T

his could prompt the system to reorder those quantities when, in fact, they’re really still on order, but have not been received yet.

The best business practice here is to add those quantities to the system using the Inventory > Adjustments > Receipts tab. Posting quantities this way will affect the purchases dollar figure for that product line / group, which is what should happen, from a business perspective. If other functions, such as Part Maintenance or even the Stock Adjustments tab in the Adjustments function, are used to post these quantities, those adjustments will impact the ‘Adjustments’ figure for the product line / group, and this figure basically represents inventory loss or gain.

The above mentioned scenario is not an inventory loss or gain. The system tracks the ‘Adjustments’ figures for all product lines & groups and prints that figure on the Line Inventory Value report (RPT027) and month end. If a store uses the above mentioned figure from this report for tax reporting purposes, they will have incorrectly reported a false inventory loss or gain. However, if they don’t use this figure, it really doesn’t matter which function they use to add these quantities.

Post by Check-in group

  1. If a Check-In Group is created and no received quantities are entered for parts in a PO for which stock receipts have been processed, system displays a window stating backordered quantities will be cleared and asks if this is the final shipment.

  • If ‘Yes’ is selected, parts with no quantity received get marked as posted (w/ none added to inventory) and on order quantities are removed.

  • If ‘No’ is selected, parts with no quantities received get marked as posted (none added to inventory) and a duplicate line is added to the PO with an open order for the same parts. Additionally, the Shipped Qty gets cleared.

  1. If a Check-In Group is created and no received quantities are entered for parts in a PO for which NO stock receipts have been processed, system marks parts with no received quantities as posted and posts none to inventory, but leaves the on order quantity.

  • Note: Best Practice is to remove the on order quantity through Part Maintenance and call support to have the Posted Date modified, or else End of Day will reset the On Order quantity and days on order.
